Command-line for Post-Assembly Profiling Module

Post-Assembly Profiling builds on the genome assembly step by focusing on how to evaluate the quality of assembled contigs and interpret their gene content. In this section, you will learn how to assess assembly reliability and then explore what genes are present within the genome. The workflow includes two main steps:

  1. Quality assessment of assemblies (e.g., completeness, contamination, N50, genome size)
  2. Characterization of gene content (antimicrobial resistance (AMR) genes, virulence factors,…)

Command-line Workflow

From now on we will work mainly on Terminal (for MacOS / Linux user) or Command Prompt (for Windows user) or Visual Studio Code application.

Access to the class server

To begin, you need to access the class setup server. Open your Terminal and connect using the following command. This will log you into the server environment where all required tools and data for the session are prepared.

ssh -i microgen2026.pem user@136.112.198.45
-------------------------------
# microgen2026.pem: key identity file (private key for authentication) 
# user: your server username
# 132.112.198.45: your server ip adress

Expore the materials available on the server

Go to your day-2/post_assembly directory using the following command:

cd /microgen2026/day-2/post-assembly/

List all files in the post_assembly directory:

ls

# -l: count number of lines

Quality Assessment of Assemblies

For this section, checkm2 is used

CheckM2 assesses the quality of genome assemblies. It estimates completeness (how much of the genome is recovered) and contamination (how much unwanted DNA is present) using a machine-learning approach based on gene content and genome features. This helps you determine whether your assembly is reliable before moving to downstream analysis.

Access conda environment

Activate the Conda environment containing the checkm2 tool:

conda activate microgen2026
-------------------------------
# conda activate: activate working enviroment 
# microgen2026: our enviroment name contains all required tools for this workshop

Run checkm2 (completeness & contamination)

Check the availability of your tool by typing its name:

checkm2

The output should look like this:

                ...::: CheckM2 v1.1.0 :::...

  General usage:
    predict         -> Predict the completeness and contamination of genome bins in a folder. Example usage: 

    checkm2 predict --threads 30 --input <folder_with_bins> --output-directory <output_folder>
    testrun         -> Runs Checkm2 on internal test genomes to ensure it runs without errors. Example usage: 

     checkm2 testrun --threads 10
    database        -> Download and set up required CheckM2 DIAMOND database for annotation

  Use checkm2 <command> -h for command-specific help.

Quality assessment of assemblies with checkm2:

checkm2 predict --threads 15 --input *.fasta --output-directory checkm2_output
-------------------------------
# checkm2 predict: run genome quality prediction 
# --threads: number of CPU cores for parallel running, high cores count speed up the analysis 
# --input: input your assembly fasta files
# --output-directory: location and file name to save results

After checkm2 finishes running, the output should look like this:

checkm2.log  diamond_output  protein_files  quality_report.tsv

Characterization of Gene Content

For this section, Kleborate is used

Kleborate characterizes Klebsiella and Escherichia coli genomes. It performs MLST typing and identifies important features such as antimicrobial resistance (AMR) genes and virulence factors. This allows you to understand the clinical and epidemiological significance of your isolates.

Access conda environment

Activate the Conda environment containing the kleborate tool:

conda activate microgen2026
-------------------------------
# conda activate: activate working enviroment 
# microgen2026: our enviroment name contains all required tools for this workshop
NoteOptional

If you are already in the conda environment microgen2026, please skip this step.

Run kleborate (version 3.1.3)

Check the availability of your tool by typing its name:

kleborate

The output should look like this:

usage: kleborate [-a ASSEMBLIES [ASSEMBLIES ...]] [-o OUTDIR] [-r] [--trim_headers] [--list_modules] [-p PRESET] [-m MODULES] [-h] [--help_all] [--version]

Kleborate: a tool for characterising virulence and resistance in pathogen assemblies

Check kleborate version:

kleborate --version

The output should look like this:

Kleborate v3.1.3

MLST typing and identification of AMR and Virulence genes using kleborate:

kleborate -p kpsc -o test_kleborate -a *.fasta -t 15
-------------------------------
# -p kpsc: specify the species complex, kpsc = klebsiella pneumoniae species complex
# -o: define the output file name
# -a: input your assembly fasta files
# -t: number of CPU cores for parallel running, high cores count speed up the analysis

After kleborate finishes running, the output should look like this:

klebsiella_pneumo_complex_output.txt

-------------------------------

/Users/Downloads
TipFor Windows users

The printout screen should look like this:

# -r: enable copy directories

# ~/microgen2026/day-2/post-assembly/: result directory on learning server
