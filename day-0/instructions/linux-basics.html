0.2 Commands & Environments

Author

Nguyen MTS

Published

May 11, 2026

Why Linux?

You might wonder why we aren’t using Windows or macOS for this. There are three main reasons:

  1. Big Data: Genomic datasets are often too large for Excel or standard text editors. And big data requires big computers, and big computers are usually Linux-based.
  2. Automation: In genomics, we often need to run the same analysis on hundreds of samples. CLI (Command Line Interface) allows us to automate this with scripts.
  3. Reproducibility: Our analyses are our commands to the Linux computer, and the Linux commands are just texts. This enables easy recording of exactly what we did, i.e. all the commands that we have run, making it easier to share and reproduce our analyses.

The interface for us to give commands to the Linux computer is called the Terminal. For many, the “black screen” of the terminal can be intimidating, but it is the most powerful tool in a bioinformatician’s toolkit.

Intro to Linux

The Terminal

The Terminal (or console) is your window into the operating system. It provides a Shell, which is the program that interprets the commands you type and tells the computer what to do. On most modern systems, the default shell is bash or zsh.

  • The Prompt: Usually looks like user@computer:~$, where:
    • user is your username.
    • computer is the hostname of the computer.
    • ~ is the current directory you are in. In this example & by default, it’s your home directory, encoded as ~, which is a shortcut for /home/user. No worries if your head spins now, you will understand this better later.
    • $ indicates that the shell is waiting for your input.
  • Case Sensitivity: Linux is case-sensitive. File.txt and file.txt are two different files.

The Command Line

A command usually follows this structure: command [options] [arguments]

Examples:

Practice
# show Linux OS name, version, and hardware architecture
hostnamectl 

# list CPU information
lscpu
Practice
echo "Hello World"
  • echo is the command (print text).
  • "Hello World" is the argument (the text to print).
Practice
ls -l microgen2026/

  • ls is the command (list files).

  • -l is an option (long format).

  • microgen2026/ is the argument (the folder to look into).

Getting Help

How can I know what a command does?

How to know what options a command has?, and what arguments it can take?

What are all the commands available on Linux?

Do I need to memorize all of them?

If you wonder about these questions, you are on the right track!

And the answer is: Yes & No. ☯️

Memorizing some common commands will definitely save you a lot of time, but memorizing all of them is unnecessary (and also un-wise).
The key is to know what is possible to do on Linux, and how to find that out.

If you forget the options or arguments of a command, or you just want to check again all of its capabilities, you can find out with the following commands:

  • command --help: Most tools have a built-in help menu (e.g., ls --help).
  • man command: Opens the “manual” page (e.g., man ls). Press q to exit.
  • info command: Opens the “info” page (e.g., info ls). Press q to exit.

and, last but not least:

  • Search & AI: Google, Stack Overflow, and AI tools are your friends when you get stuck! Lots of time there are no “manual” or “info” pages locally on the computer, or they are not user-friendly enough, so to check the online resources is a must!
TipHundreds to Thousands

Do you know that, there are up to 150–300 commands in the core utilities of Linux, depending on the OS distribution? And if we count all executable files that are later installed, the number can be in the thousands!

The key, therefore, is actually our imagination and creativity. Just exploit the Search & AI tools out there to find the right commands for your needs.

The File System

Think of the Linux file system as a giant tree.

  • Root (/): The base of the entire system.

  • Home Directory (~, or $HOME): Your personal space where you have full control.

  • Path: The “address” of a file or folder.

    • Absolute Path: Starts from the root (e.g., /home/user/data/sample.fastq).

    • Relative Path: Starts from where you currently are (e.g., data/sample.fastq).

Navigation in the File System

  • pwd (Print Working Directory): “Where am I?”
  • ls: “What’s in here?”
    • ls -F: Adds a / to directories.
    • ls -l: Shows detailed information (permissions, size, etc.).
    • ls -h: Human-readable sizes (KB, MB, GB).
  • cd: “Go somewhere else.”
    • cd ..: Move up one level.
    • cd ~: Go back home.

Conventions:

  • . and $PWD: refer to the current “working” directory.
  • ..: refers to the parent directory of the current directory.
  • ~ and $HOME: refer to the home directory.
Practice
# Check where you are
pwd
# other way
echo $PWD

# See what files are in the current directory
ls

# Go to the data directory for the workshop
cd /microgen2026/

# List all files in the current directory, including hidden files
ls -a

# Go to day-0 directory
cd day-0

# List files in the current directory in long format with human-readable sizes and show hidden files
ls -lah

# Go to home directory
cd ~

# Go back to the day-0 directory
cd -

# Go back to the parent directory `microgen2026`
cd ..
TipTips

When you move around, i.e. changing the current directory, did you notice any change to the prompt, particularly the part following the user@computer:?

Tab-completion is your best friend!
When you start typing a command, file, or directory name, press the Tab key to auto-complete it. If there are multiple possibilities, press Tab twice to see all of them.

Files in Linux

Anatomy of a File Name

In Windows, the file extension (.docx, .exe) tells the system what the file is. In Linux, extensions are mostly for humans. Linux determines what a file is by looking at its content (specifically “magic bytes” at the beginning), not its name.

You can name a file anything:

  • genome.fasta (Helpful)
  • ingredients_list.banhmi (Valid)
  • data_source.nuocmam (Valid)

However, we usually keep standard extensions so we don’t confuse ourselves or our colleagues!

Types of Files (in Genomics)

  1. Text Files: Plain text you can read. Examples: .fasta, .fastq, .csv, .sh.
  2. Compressed Files: To save space, we often zip files. Common extensions: .gz, .zip, .tar.gz. Most Linux tools can “peek” inside .gz files without unzipping them.
  3. Binary Files: Not human-readable. These are programs or optimized data formats (e.g., compiled tools, .bam or .bcf files).
  4. Executable Files: Files that can be run as programs. Examples: .sh, .py, .R.

Peeking into File Contents

We rarely open huge genomics files in editors. We use these instead:

  • cat: Print the whole file (only for small files!).
  • head: Show the first few lines.
  • tail: Show the last few lines.
  • less: The “gold standard” for viewing large files. Use arrows to scroll, / to search, and q to quit.
Practice
# Print the whole file
cat README.md

# Show the first 10 lines
head -n 10 README.md

# Show the last 5 lines
tail -n 5 README.md

# Less is More
less README.md

Standard Input, Output, and Error

  • Standard In (stdin): Where the command receives its input (the keyboard or redirection).
  • Standard Out (stdout): Where the command sends its regular results (the screen).
  • Standard Error (stderr): Where the command sends error messages.

Redirection

Redirection allows you to change where a command gets its input or sends its output.

  • command < file: Redirect contents of a file as input to a command.
  • command > file: Save output (stdout) to a file (overwrites).
  • command >> file: Append output (stdout) to a file (adds to the end).
  • command 2> file: Redirect error messages (stderr) to a file.
  • command > file 2>&1: Redirect both stdout and stderr to a single file.

Example: Save the list of files to a text file.

Practice
ls -lh > file_list.txt

Creating & Managing Files

  • mkdir: Create a new directory (which is just a special type of file!).
    • mkdir linux_basics_practice
  • touch: Create an empty file.
    • touch learning_notes.txt
  • nano: Edit a file.
    • nano learning_notes.txt
    • Ctrl + O to save
    • Ctrl + X to exit
  • cp: Copy files or directories.
    • cp source.txt destination.txt
    • cp -r source_dir destination_dir
  • mv: Move or rename files & directories.
    • mv old_name.txt new_name.txt
    • mv source_dir destination_dir
NoteTip

Referring to files by patterns: - *: matches any sequence of characters - ?: matches any single character - [abc]: matches any one of the characters a, b, or c - [a-z]: matches any one of the characters from a to z

Text Analysis & Manipulation

grep: Search for a pattern

Search for strings or patterns within files.

  • -i: Ignore case (search for both DNA and dna).
  • -v: Invert match (show lines that do not contain the pattern).
  • -c: Count the number of matching lines.
  • -n: Show line numbers.
  • -w: Match whole words only.
Practice
# Search for "Human Microbiome Project (HMP)" in the metadata file
grep "Human Microbiome Project (HMP)" crAssphage_in_public.csv

# Count how many lines contain "PASS" in the VCF file
grep -c "PASS" 965Kp.vcf

wc: Count lines, words, and characters

  • -l: Count lines.
  • -w: Count words.
  • -c: Count characters (bytes).
Practice
# Count how many lines are in the VCF file
wc -l 965Kp.vcf

cut: Cut out sections from each line of files

Used to extract specific columns from text files.

  • -f: Select fields (columns).
  • -d: Define the delimiter (the default is tab).
Practice
# Extract the 2nd and 4th column from a tab-delimited file
cut -f 2,4 metadata_carbResist.tsv

# Extract the 5th column from a CSV file (comma delimited)
cut -f 5 -d "," crAssphage_in_public.csv

sort: Sort lines of text files

  • -n: Numeric sort.
  • -r: Reverse order.
  • -k: Sort by a specific column (key).
  • -t: Specify the delimiter (the default is tab).
  • -u: Unique (remove duplicates).
Practice
# Sort the metadata by the 3rd column (Gbp) numerically in reverse
sort -k 2 -n -r genome_sizes.tsv

tr: Translate or delete characters

Practice
# first, let see the original messy file
cat messy_data.csv

# Convert a CSV to a TSV (comma to tab)
tr ',' '\t' < messy_data.csv

# Convert all text to uppercase
tr '[:lower:]' '[:upper:]' < messy_data.csv

sed: Stream editor for text manipulation

A powerful stream editor for filtering and transforming text. Most commonly used for find-and-replace using the syntax: sed 's/old/new/g'.

Practice
# Change all filenames starting with 'sample_' to 'sample-' in our messy file
sed 's/sample_/sample-/g' messy_data.csv

# Put the change to a new file
sed 's/sample_/sample-/g' messy_data.csv > pretty_data.csv

# or, to make the changes permanent on the file, use -i option

Building Up Complexity

Pipes (|)

The pipe takes the output of one command and sends it as input to another. This allows you to build complex workflows by chaining simple commands.

Example: Ignore all the header lines that starts with # and extract the 2nd, 4th, 5th, and 7th columns from 965Kp.vcf:

Example
grep -v "^#" 965Kp.vcf | cut -f2,4,5,7

# then count the number of lines
grep -v "^#" 965Kp.vcf | cut -f2,4,5,7 | wc -l

Variables

Variables store information (text, numbers, filenames, etc.) for later use.

Example
MY_SAMPLE="SRR24679994"

# Print the value of the variable
echo $MY_SAMPLE

# Use the variable in a command - Find the sample in the metadata file
grep -w "$MY_SAMPLE" metadata_carbResist.tsv

Types of variables:

  • Local variables: Most of the variables you set in your shell session only exist in that session, which means you cannot find them in new terminal sessions. These are local. If you want to reuse them, you will have to specify them every time you open a new terminal session.
  • Global variables: These are variables that exist “everywhere”. Examples of global variables at the heart of a Linux system are: $HOME, $PWD, $USER, etc.

Bash Script - A Flow of Commands

An example script below will download the crAssphage reference genome from NCBI.

# View the content of the script
cat download_crAssphage_genome.sh
download_crAssphage_genome.sh
#!/bin/bash
# Download the crAssphage - Carjivirus communis - reference genome from NCBI

# Create a directory to store the genome
mkdir -p crAssphage 

# Download the genome sequence in FASTA format
wget "https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi?db=nucleotide&id=NC_067194.1&rettype=fasta&retmode=text" -O crAssphage/NC_067194.1.fasta

# Download the genome annotation in GenBank format
wget "https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi?db=nucleotide&id=NC_067194.1&rettype=gb&retmode=text" -O crAssphage/NC_067194.1.gbk

# List the downloaded files
ls -lh crAssphage/
# Run the script by calling the bash interpreter
bash download_crAssphage_genome.sh

# Make the script executable
chmod +x download_crAssphage_genome.sh

# Run the script by calling the script directly
./download_crAssphage_genome.sh

Software Environments: Conda & Mamba

Different bioinformatics tools often require different versions of the same software. If you install everything together, they might clash. We use Mamba (the faster version of Conda) to create isolated “bubbles” or environments for each tool.

Example: Installing Kingfisher - a fast downloading tool for public sequence files and their metadata annotations.

Example
# Create an environment for the kingfisher program
mamba create -n kingfisher -c conda-forge -c bioconda kingfisher

# Enter the bubble
mamba activate kingfisher

# Get the program's help message
kingfisher

# Leave the bubble
mamba deactivate

Self Exploration

Once you are comfortable with the basics, you can further power up your Linux skills with these four more “magical spells” 🪄:

tmux & screen

Bioinformatics jobs are often computationally heavy and can take hours or days to run. If your internet suddenly disconnects, the job on the big remote computer might die, and you will lose all your progress. Also, you cannot let your laptop open for days on end.

So when running on remote computers, like our Google Cloud instance, tmux & screen are handy tools that allow you to run “sessions” that keep going even if you log out.

Some useful tips for using those tools:

  • tmux: The modern standard for terminal multiplexing.
    • tmux new -s <name>: Start a new session.
    • Ctrl+b then d: Detach from the session.
    • tmux ls: List active sessions.
    • tmux attach -t <name>: Reattach to a session.
    • Manual Page
  • screen: A classic alternative available on most systems.
    • screen -S <name>: Start a new session.
    • Ctrl+a then d: Detach from the session.
    • screen -ls: List active sessions.
    • screen -r <name>: Reattach to a session.
    • Manual Page

Loops

Automate tasks across hundreds of samples.

Example
for file in /microgen2026/day-0/*.fasta; do 
    echo "Processing $file..."; 
    grep -c '>' $file;
done

GNU parallel

Run many commands at once using multiple CPU cores. A true game-changer for processing large cohorts.

Example
# Get the number of contigs in all samples parallelly on 4 CPU cores
ls day-0/*.fasta | parallel -j 4 grep -c '>' {}
