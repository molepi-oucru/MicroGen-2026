0.1.7 Troubleshooting and FAQs

Author

Minh-Quan Ton-Ngoc

Published

May 11, 2026

1 Troubleshooting

python is not recognized on Windows

Likely cause: Python was installed without adding it to PATH.

Simple fix:

  1. Open the Python installer again.
  2. Choose Modify or reinstall Python.
  3. Tick Add python.exe to PATH if shown.
  4. Close and reopen PowerShell.
  5. Try:
python --version

Ask an instructor if Windows still cannot find Python.

pip is not recognized

Likely cause: Windows can find Python but not the separate pip command.

Simple fix: use python -m pip instead:

python -m pip --version
python -m pip install numpy pandas

On Ubuntu, use python3 -m pip:

python3 -m pip --version
python3 -m pip install numpy pandas

If pip is missing on Ubuntu, install it:

sudo apt update
sudo apt install python3-pip python3-venv

Ask for help if python -m pip also fails.

R package installation fails

Likely causes:

  • internet connection problem,
  • CRAN mirror problem,
  • missing system build tools,
  • BiocManager has not been installed yet for Bioconductor packages,
  • or an older package binary is not available.

Simple fixes:

  1. Try again with a different CRAN mirror.
  2. Restart R or RStudio.
  3. If the error mentions BiocManager, install it first:
if (!requireNamespace("BiocManager", quietly = TRUE)) {
  install.packages("BiocManager")
}
  1. On Windows, if the error mentions compilers, make, gcc, g++, gfortran, or Rtools, install Rtools45 from the official CRAN Rtools page:
https://cran.r-project.org/bin/windows/Rtools/
  1. On macOS, install Xcode Command Line Tools:
xcode-select --install
  1. On Ubuntu, install common build dependencies:
sudo apt update
sudo apt install build-essential gfortran libcurl4-openssl-dev libssl-dev libxml2-dev

Ask an instructor if the error continues after these steps, especially if it mentions Fortran, gfortran, unavailable binary packages, or incompatible Bioconductor versions.

phyloseq installation fails

Likely cause: phyloseq comes from Bioconductor, not ordinary CRAN, so it needs BiocManager.

Simple fix:

if (!requireNamespace("BiocManager", quietly = TRUE)) {
  install.packages("BiocManager")
}

BiocManager::install("phyloseq", ask = FALSE, update = TRUE)

Ask an instructor if Bioconductor says your R version is not compatible.

Windows says Rtools is required

Likely cause: R needs to build one or more packages from source.

Simple fix:

  1. Install Rtools45 from the official CRAN page:
https://cran.r-project.org/bin/windows/Rtools/
  1. Keep the default installation folder:
C:\rtools45
  1. Restart R or RStudio.
  2. Check that R can find Rtools:
install.packages("pkgbuild")
pkgbuild::has_rtools(debug = TRUE)
  1. Retry the failed package installation.

Ask an instructor for help if pkgbuild::has_rtools(debug = TRUE) does not return TRUE.

macOS asks for command line tools

Likely cause: an R package needs to compile code.

Simple fix:

xcode-select --install

Wait for the installer to finish, then reopen R or RStudio.

macOS cannot verify that this app is free from malware

Likely cause: macOS Gatekeeper blocked an app downloaded outside the App Store. This can happen with scientific tools such as Artemis, BandageNG, or SeaView.

1.0.1 Fix option 1: manual Privacy & Security method

Use this method first if you are not comfortable with Terminal.

  1. Open Finder and locate the app you want to open.
  2. Double-click the app icon. macOS should show the security warning again.
  3. Open the Apple menu.
  4. Go to System Settings > Privacy & Security.
  5. Under Allow apps downloaded from, look for the notification about the blocked application.
  6. Click Open Anyway.
  7. Enter your admin username and password if macOS asks.
  8. Open the app again. macOS should remember this choice for that app.

1.0.2 Fix option 2: Terminal quarantine removal

Use this only for an app you intentionally downloaded from an official source.

For an app in /Applications, replace AppName.app with the real app name:

xattr -dr com.apple.quarantine "/Applications/AppName.app"

For an app in Downloads:

xattr -dr com.apple.quarantine "$HOME/Downloads/AppName.app"

Then try opening the app again.

Do not disable SIP for workshop setup. Some web guides discuss turning off Gatekeeper globally or disabling System Integrity Protection. Do not do this for the workshop. Use Open Anyway or remove quarantine only for the specific app you trust.

If you are still unsure, ask an instructor before changing security settings.

Background references for instructors:

  • https://maclife.co/huong-dan-tat-gatekeeper-bat-tuy-chon-anywhere-tren-macos-sierra-10-12-de-cai-ung-dung-ben-ngoai-app-store.html
  • https://maclife.co/huong-dan-tat-system-integrity-protection-sip-de-cai-app.html
VS Code cannot connect to the server

Likely causes:

  • wrong server IP,
  • wrong username,
  • wrong key path,
  • key file was moved,
  • server not ready,
  • or local network blocks SSH.

Simple checks:

  1. Try the same SSH command in PowerShell or Terminal.
  2. Check that the key file exists at the path in your SSH config.
  3. Check that HostName, User, and IdentityFile are correct.

Ask the admin if ordinary SSH fails too.

SSH says Permission denied

Likely causes:

  • wrong username,
  • wrong private key,
  • private key not selected,
  • or the admin has not added your account yet.

Simple fix: check the command carefully:

ssh username@SERVER_IP -i /path/to/key.pem

Ask the admin if the username, server IP, and key are correct but the error continues.

SSH says Bad permissions

This is common on macOS when the key file is readable by other users.

Fix:

chmod 600 /path/to/key.pem

Then connect again.

PEM key path contains spaces

On Windows, put quotation marks around the key path:

ssh username@SERVER_IP -i "C:\Users\Your Name\Downloads\microgen2026.pem"

On macOS, either quote the path or move the key to a simpler location:

ssh username@SERVER_IP -i "/Users/Your Name/Downloads/microgen2026.pem"
MobaXterm cannot open GUI windows

Likely causes:

  • the MobaXterm X server is not running,
  • X11 forwarding is not enabled,
  • or the remote program cannot find DISPLAY.

Simple checks inside the server SSH session:

echo $DISPLAY
xclock

If xclock does not open, use VNC or ask an instructor to check the MobaXterm session settings.

X11 forwarding does not work

Likely causes:

  • XQuartz is not running on macOS,
  • VcXsrv is not running on Windows,
  • the SSH session was opened without -Y,
  • or network latency makes X11 unstable.

Simple fix on macOS:

ssh -Y username@SERVER_IP -i /path/to/key.pem

Simple fix on Ubuntu desktop:

ssh -Y username@SERVER_IP -i /path/to/key.pem
echo $DISPLAY
xclock

If you are using Ubuntu through WSL, a VM, or a headless machine, use VNC if X11 is difficult to configure.

If it remains unreliable, use VNC.

VNC tunnel connects but the screen is blank

Likely causes:

  • you connected to the wrong port,
  • the tunnel window was closed,
  • VNC was not started on the server,
  • or the remote desktop is locked.

Simple checks on the server:

list-vnc
fix-vnc-lock

If needed, stop your display and start again:

stop-vnc 2
start-vnc

Replace 2 with your actual display number.

Java is not found

Likely cause: Java was not installed or the terminal has not refreshed its PATH.

Simple fixes:

  1. Install OpenJDK 17. On Windows, prefer Temurin OpenJDK 17 LTS.
  2. Close and reopen PowerShell or Terminal.
  3. Run:
java -version

On Ubuntu, install OpenJDK 17 with:

sudo apt update
sudo apt install openjdk-17-jdk
java -version

Ask for help if Java is installed but still not found.

RStudio cannot find R

Likely cause: RStudio was installed before R, or R installation did not finish.

Simple fixes:

  1. Install R 4.5.0.
  2. Restart the computer.
  3. Open RStudio again.

If RStudio still cannot find R, ask an instructor to check the R installation path.

2 FAQs

This section answers common setup questions in the way they often appear during installation.

Do I need to install Rtools45 before class?

Usually no. Install Rtools45 only if you are on Windows and R reports that it needs to build a package from source.

You probably need Rtools45 if you see errors mentioning:

  • Rtools
  • make
  • gcc
  • g++
  • gfortran
  • compilation failed
  • package ... is only available in source form

If you use Windows with R 4.5.0, install Rtools45 from:

https://cran.r-project.org/bin/windows/Rtools/

Keep the default location:

C:\rtools45

Then restart R or RStudio and retry the package installation.

I installed Rtools45. Why does R still not find it?

Likely cause: R or RStudio was already open before Rtools45 was installed.

Simple fix:

  1. Close all R, RGui, and RStudio windows.
  2. Reopen R or RStudio.
  3. Run:
install.packages("pkgbuild")
pkgbuild::has_rtools(debug = TRUE)

If it still does not return TRUE, check that Rtools45 was installed in C:\rtools45. Ask an instructor if your institution’s laptop security policy blocks the installer.

Should macOS users install Rtools?

No. Rtools is for Windows only.

On macOS, if R needs to compile a package, install Apple’s Command Line Tools:

xcode-select --install

After the installer finishes, close and reopen R or RStudio.

Should Ubuntu users install Rtools?

No. Rtools is for Windows only.

On Ubuntu, if R needs to compile a package, install build tools and common development libraries:

sudo apt update
sudo apt install build-essential gfortran libcurl4-openssl-dev libssl-dev libxml2-dev

After installation, close and reopen R.

R says there is no package called 'BiocManager'. What should I do?

Install BiocManager first:

install.packages("BiocManager")

Then install the Bioconductor package:

BiocManager::install("phyloseq", ask = FALSE, update = TRUE)

BiocManager is not included with a fresh R installation, so this message is normal on a new computer.

R asks Do you want to install from sources the package which needs compilation?

For beginners, choose No if R also offers a binary package. Binary packages are easier to install.

If R cannot offer a binary package and says source installation is required:

  • Windows: install Rtools45, restart R or RStudio, and try again.
  • macOS: install Xcode Command Line Tools, restart R or RStudio, and try again.
  • Ubuntu: install build-essential, gfortran, and the relevant lib*-dev packages, restart R, and try again.

Ask an instructor if the package is still failing after the build tools are installed.

R asks me to update many packages. Should I say yes?

Usually yes, but it can take time. If you are in the workshop room and the internet is slow, ask an instructor first.

For the phyloseq command in this guide, we use:

BiocManager::install("phyloseq", ask = FALSE, update = TRUE)

This allows Bioconductor to update packages needed for compatibility.

I see package is not available for this version of R.

Likely causes:

  • you installed the wrong R version,
  • the package repository was temporarily unavailable,
  • or R is using a stale package mirror.

Simple checks:

R.version.string

You should see R 4.5.0 for this workshop. If the version is different, ask an instructor whether to reinstall R 4.5.0 or continue with your current version.

Python installed, but python --version does not work on Windows.

Likely cause: Python was installed without adding it to PATH.

Simple fix:

  1. Re-run the Python installer.
  2. Choose Modify or reinstall.
  3. Tick Add python.exe to PATH.
  4. Close and reopen PowerShell.
  5. Run:
python --version

If that still fails, try:

py --version
pip install numpy pandas fails.

Use Python’s module form of pip, which is more reliable:

python -m pip install --upgrade pip
python -m pip install numpy pandas

On macOS:

python3 -m pip install --upgrade pip
python3 -m pip install numpy pandas

If the error mentions internet access, try a different network or ask an instructor.

VS Code Remote SSH asks me to choose Linux, Windows, or macOS for the server.

Choose Linux. The workshop server is a Linux server.

VS Code Remote SSH keeps asking for a password.

Likely causes:

  • VS Code cannot find your .pem key file,
  • the key path in the SSH config is wrong,
  • or the server username is wrong.

Open your SSH config and check:

Host microgen2026
    HostName SERVER_IP
    User username
    IdentityFile /path/to/key.pem

On Windows, a key path may look like:

IdentityFile C:\Users\YourName\Downloads\microgen2026.pem

Ask the admin if the username, server IP, and key path are all correct but login still fails.

SSH says WARNING: UNPROTECTED PRIVATE KEY FILE.

This usually happens on macOS or Linux when the .pem file permissions are too open.

Run:

chmod 600 /path/to/key.pem

Then try SSH again.

MobaXterm opens SSH, but graphical windows do not appear.

Check X11 first:

echo $DISPLAY
xclock

If xclock does not appear, check that MobaXterm’s X server is running and that X11 forwarding is enabled in the session settings.

If X11 remains slow or unreliable, use VNC instead.

VNC asks for a password. Is that my server password?

No. It is the VNC password you created when you ran:

start-vnc

It is separate from your .pem key and separate from any Linux password.

If you forgot the VNC password, ask an instructor before resetting it.

The VNC port is not 5901. Is that a problem?

No. Use the port printed by start-vnc, for example 5902 or 5903.

If start-vnc prints:

VNC port: 5902

then your tunnel should use 5902:

ssh -L 5902:localhost:5902 username@SERVER_IP -i /path/to/key.pem
Java is optional. Should I install it now?

Install Java only if your instructor asks you to use Artemis or another Java-based graphical tool.

For this workshop, use OpenJDK 17. On Windows, prefer Temurin OpenJDK 17 LTS. After installing, check:

java -version

If you are short on time before class, prioritize R, Python, VS Code, and server access first.
