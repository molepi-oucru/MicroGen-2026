1 Ubuntu setup

Use this section if you are setting up your own Ubuntu laptop, WSL with graphical support, or an Ubuntu virtual machine.

Warning

Do not install system packages on the workshop server or course instance unless an instructor or admin explicitly tells you to. The workshop server is shared and already has the required teaching environment prepared.

1.1 Install Visual Studio Code

The official VS Code documentation recommends downloading the .deb package for Debian/Ubuntu systems from https://code.visualstudio.com/download.

  1. Download the .deb package for Ubuntu.
  2. Open a terminal in the folder where the file was downloaded.
  3. Install it with:
sudo apt install ./code_*.deb

Open VS Code and install these extensions:

  • Remote - SSH by Microsoft
  • Python by Microsoft
  • R by REditorSupport
  • Quarto by Quarto
Recommended shortcut: import the MicroGen2026 VS Code profile

After installing VS Code, you can download the workshop profile file: Microgen2026.code-profile.

To import it, open VS Code and choose Manage > Profiles > Profiles > More Actions… (dropdown icon) > Import Profile…, then select the downloaded Microgen2026.code-profile file.

If the profile imports successfully, it replaces the manual VS Code extension setup above and adds workshop-specific settings/customizations. If the profile import does not work on your computer, install the extensions manually using the list above.

1.2 Install Python and packages

Ubuntu may provide a Python version through apt, but the workshop target is Python 3.13. If your Ubuntu release does not provide Python 3.13 directly, use the official Python source release or ask an instructor before class.

For a simple local setup, install Python tools and then check the version:

sudo apt update
sudo apt install python3 python3-pip python3-venv
python3 --version
python3 -m pip --version

Install the Python packages:

python3 -m pip install --upgrade pip
python3 -m pip install numpy pandas
python3 -c "import numpy, pandas; print('Python packages OK')"

1.3 Install R 4.5.x from CRAN

CRAN provides Ubuntu packages for the current R 4.5 release series on supported Ubuntu LTS releases. Follow the official CRAN Ubuntu page if these commands need updating: https://cran.r-project.org/bin/linux/ubuntu/.

sudo apt update -qq
sudo apt install --no-install-recommends software-properties-common dirmngr wget ca-certificates
wget -qO- https://cloud.r-project.org/bin/linux/ubuntu/marutter_pubkey.asc | sudo tee -a /etc/apt/trusted.gpg.d/cran_ubuntu_key.asc
sudo add-apt-repository "deb https://cloud.r-project.org/bin/linux/ubuntu $(lsb_release -cs)-cran40/"
sudo apt install --no-install-recommends r-base r-base-dev
R --version

Install the required R packages from inside R:

install.packages(c(
  "tidyverse",
  "dplyr",
  "tidyr",
  "readr",
  "ggplot2",
  "ggrepel",
  "vegan",
  "patchwork",
  "scales",
  "data.table",
  "broom",
  "knitr",
  "BiocManager"
))

BiocManager::install("phyloseq")

1.4 Install OpenJDK 17

sudo apt install openjdk-17-jdk
java -version

The remaining Ubuntu commands below are mainly for optional graphical applications.

Use this section only if you are setting up an Ubuntu laptop, WSL with graphical support, or an Ubuntu virtual machine. These commands require administrator access through sudo.

Workshop server note

Learners usually should not install system packages on the workshop server. Use this Ubuntu section for your own Ubuntu computer or VM unless an admin tells you otherwise.

Update the package index:

sudo apt update

Install OpenJDK 17, Artemis, NCBI BLAST+, and SeaView from Ubuntu packages:

sudo apt install openjdk-17-jdk artemis ncbi-blast+ seaview

Check the tools:

java -version
art
blastn -version
makeblastdb -version
seaview

If art does not start Artemis on your Ubuntu version, try:

artemis

Install BandageNG with a conda-compatible package manager if you use conda, mamba, micromamba, or pixi:

conda create -n bandageng -c conda-forge -c bioconda bandage_ng
conda activate bandageng
BandageNG

If the executable name is different in your environment, list matching commands:

compgen -c | grep -i bandage
Ubuntu GUI requirement

Artemis, BandageNG, and SeaView are graphical apps. They need a desktop session, X11, VNC, or another working display system. On a headless server, install and use them only if your instructor or admin has configured graphical access.
