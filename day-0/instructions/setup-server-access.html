0.1.5 Workshop server access

Author

Minh-Quan Ton-Ngoc

Published

May 11, 2026

1 Server access setup

This section is adapted from the MicroGen 2026 instructor access guide. Your exact server IP address, username, and key file will be provided by the workshop admin.

1.1 What information you will receive from the admin

You should receive:

  • your username,
  • the server IP address,
  • a .pem private key file,
  • and, if relevant, your assigned instance name.
Private key safety

Do not share the .pem key. Do not paste it into this notebook, a chat window, email, or a shared document.

1.2 SSH from Windows PowerShell

Open PowerShell and replace username, SERVER_IP, and the key path with your real details:

ssh username@SERVER_IP -i "C:\path\to\key.pem"

The quotation marks are important if the path contains spaces.

After login, check your server location:

whoami
pwd
ls -lh /workspace/$USER /scratch/$USER

Use /workspace/<username> for exercises and outputs you want to keep. Use /scratch/<username> for large temporary files.

1.3 SSH from macOS Terminal

Before your first login, fix the key file permissions:

chmod 600 /path/to/key.pem

This tells macOS that only you can read the key file.

Connect:

ssh username@SERVER_IP -i /path/to/key.pem

After login:

whoami
pwd
ls -lh /workspace/$USER /scratch/$USER

1.4 SSH from Ubuntu Terminal

The Ubuntu command is the same as macOS. First, protect your key file:

chmod 600 /path/to/key.pem

Connect:

ssh username@SERVER_IP -i /path/to/key.pem

After login:

whoami
pwd
ls -lh /workspace/$USER /scratch/$USER

1.5 VS Code Remote SSH

  1. Open VS Code.
  2. Install the Remote - SSH extension.
  3. Open the Command Palette:
    • Windows: Ctrl+Shift+P
    • macOS: Command+Shift+P
    • Ubuntu/Linux: Ctrl+Shift+P
  4. Choose Remote-SSH: Add New SSH Host.
  5. Paste your SSH command, for example:
ssh username@SERVER_IP -i /path/to/key.pem
  1. Choose the SSH configuration file suggested by VS Code.
  2. Open the Command Palette again.
  3. Choose Remote-SSH: Connect to Host.
  4. Select your new host.
  5. Open your assigned workspace folder:
/workspace/<username>

Do not use /databases as your main VS Code workspace. Shared database files are usually read-only.

1.5.1 Example SSH config

Windows example:

Host microgen2026
    HostName SERVER_IP
    User username
    IdentityFile C:\path\to\key.pem

macOS example:

Host microgen2026
    HostName SERVER_IP
    User username
    IdentityFile /path/to/key.pem

Ubuntu example:

Host microgen2026
    HostName SERVER_IP
    User username
    IdentityFile /path/to/key.pem
Long-running commands

If you run a long analysis on the server, use tmux so the command can continue if your network drops:

tmux new -s microgen

Detach with Ctrl-b, then press d. Return later with:

tmux attach -t microgen

1.6 X11 forwarding

X11 opens a single graphical program from the server on your laptop. For example, the fastqc window can appear on your own screen while the program runs on the server.

1.6.1 Windows: MobaXterm recommended

MobaXterm includes an X server.

  1. Open MobaXterm.
  2. Check that the X server is running.
  3. Create or open your SSH session.
  4. Make sure X11 forwarding is enabled in the SSH session settings.
  5. In the server terminal, run:
echo $DISPLAY
xclock

If a clock window opens, X11 is working. You can then try:

fastqc

1.6.2 Windows: VcXsrv alternative

If you use VcXsrv instead of MobaXterm:

  1. Install VcXsrv.
  2. Open XLaunch.
  3. Choose:
Multiple windows
Start no client
Clipboard enabled
Disable access control

PowerShell uses this syntax:

$env:DISPLAY = "127.0.0.1:0.0"
echo $env:DISPLAY
ssh -Y username@SERVER_IP -i "C:\path\to\key.pem"

Inside the SSH session:

echo $DISPLAY
xclock
fastqc
PowerShell is not CMD

Do not use set DISPLAY=... in PowerShell. Use $env:DISPLAY = "127.0.0.1:0.0".

1.6.3 macOS: XQuartz

macOS requires XQuartz for X11.

  1. Install XQuartz from https://www.xquartz.org/.
  2. Open XQuartz once.
  3. Close and reopen Terminal.
  4. Connect with trusted X11 forwarding:
ssh -Y username@SERVER_IP -i /path/to/key.pem

Test:

echo $DISPLAY
xclock
fastqc

If the window does not appear, make sure XQuartz is running. If X11 is slow, use VNC instead.

1.6.4 Ubuntu: native X11 desktop or VNC

If you are using a normal Ubuntu desktop session, X11 is usually already available locally. Connect with trusted X11 forwarding:

ssh -Y username@SERVER_IP -i /path/to/key.pem

Test:

echo $DISPLAY
xclock
fastqc

If you are using WSL, a virtual machine, or a headless Ubuntu system, graphical windows may need extra configuration. In that case, VNC is usually easier than X11.

1.7 VNC access

VNC opens a full remote desktop. It is often more stable than X11 for graphical tools.

Use VNC only through SSH tunnel

Do not connect directly to a public VNC port. Start VNC on the server, create an SSH tunnel, then connect your VNC client to localhost.

1.7.1 Start VNC on the server

First, SSH to the server. Then run:

start-vnc

The first time, it asks you to create a VNC password.

Important:

  • this is a separate VNC password,
  • it is not your Linux password,
  • it is not your private key,
  • when asked for a view-only password, type n.

The command prints something like:

Display: :2
VNC port: 5902

Write down the VNC port. The port may be 5901, 5902, 5903, or another number.

1.7.2 Generic SSH tunnel

Open a new terminal window on your laptop and keep your original SSH session open. Replace 5902 with the port printed by start-vnc:

ssh -L 5902:localhost:5902 username@SERVER_IP -i /path/to/key.pem

Keep this tunnel window open.

Then connect your VNC client to:

localhost:5902

If your VNC port is different, use that number instead.

1.7.3 Windows VNC with MobaXterm

In MobaXterm:

  1. Start VNC on the server with start-vnc.
  2. Open Tunneling > New SSH tunnel.
  3. Choose local port forwarding.
  4. Fill in:
Forwarded port: your VNC port
Remote server: localhost
Remote port: your VNC port
SSH server: SERVER_IP
SSH login: username
SSH port: 22
Use private key: key.pem

Example if start-vnc reports port 5902:

Forwarded port: 5902
Remote server: localhost
Remote port: 5902
SSH server: SERVER_IP
SSH login: username
SSH port: 22
Use private key: key.pem

Start the tunnel and keep it running.

Then create a VNC session:

Remote hostname: localhost
Port: your VNC port

Enter the VNC password you created with start-vnc.

1.7.4 macOS VNC

macOS includes Screen Sharing.

  1. Start VNC on the server with start-vnc.
  2. Open a new Terminal window and create the SSH tunnel:
ssh -L 5902:localhost:5902 username@SERVER_IP -i /path/to/key.pem
  1. In Finder, choose:
Go > Connect to Server
  1. Enter:
vnc://localhost:5902
  1. Enter the VNC password you created with start-vnc.

You can also use RealVNC Viewer or TigerVNC Viewer if Screen Sharing does not work well on your Mac.

1.7.5 Ubuntu VNC

On Ubuntu, install a VNC client such as TigerVNC Viewer or Remmina:

sudo apt update
sudo apt install tigervnc-viewer remmina

Start VNC on the server with start-vnc, then create the SSH tunnel from a new Ubuntu Terminal window:

ssh -L 5902:localhost:5902 username@SERVER_IP -i /path/to/key.pem

Open your VNC client and connect to:

localhost:5902

Use the VNC password you created when running start-vnc.

1.7.6 Stop VNC when finished

List your VNC sessions:

list-vnc

If the display is :2, stop it with:

stop-vnc 2

If the display is :1, stop it with:

stop-vnc 1

If the remote desktop is locked, run:

fix-vnc-lock
list-vnc

Then restart your own VNC display.
